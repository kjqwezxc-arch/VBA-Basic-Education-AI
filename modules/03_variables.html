Variables & Data Types

Learn how to store, manage, and use data in your VBA programs

What is a Variable?

A variable is a named container in memory that holds a value. Think of it like a labeled box:

  • You declare a variable (create the box)
  • You assign a value (put something in the box)
  • You use the variable (take the value out)
  • The value can change while the macro runs
📌 Why use variables? Store user input, calculation results, loop counters, file paths, sheet names — anything that can change.

Declaring Variables — The Dim Statement

Dim (short for Dimension) is used to declare variables:

' Syntax: Dim variableName As DataType Dim firstName As String ' text Dim age As Integer ' whole numbers -32768 to 32767 Dim salary As Double ' decimal numbers Dim isActive As Boolean ' True or False Dim birthDate As Date ' date values ' Assign values using = firstName = "Yamaha" age = 30 salary = 450000.50 isActive = True birthDate = #1/15/1990# ' Dates use # delimiters
📐 Declare before use! Always declare variables at the top of the Sub before using them. This is good practice and required when using Option Explicit.

Option Explicit — Always Use It!

Option Explicit forces you to declare all variables before using them.

✅ Why it saves you Without it, typos create new variables silently, causing bugs that are nearly impossible to find. Option Explicit catches them immediately.
' Add this at the very TOP of every module: Option Explicit Sub GoodPractice() Dim total As Long total = 100 ' totla = 50 ← Error! (typo caught!) End Sub
❌ Without Option Explicit
Sub BadPractice() total = 100 ' Typo below - creates NEW variable! totla = 50 ' ← Silent bug! MsgBox total ' Still shows 100 End Sub
💡 Auto-Add Option Explicit In VBE: Tools → Options → Editor tab → Check "Require Variable Declaration"

Constants — Values That Never Change

Use Const for values that are fixed throughout your program:

Option Explicit ' Module-level constants (available to all subs in this module) Const TAX_RATE As Double = 0.10 Const COMPANY_NAME As String = "Yamaha Motor Co." Const MAX_ROWS As Long = 1048576 Sub CalculateTax() Dim price As Double Dim tax As Double price = 50000 tax = price * TAX_RATE ' Uses the constant MsgBox "Tax = ¥" & tax End Sub
📌 Constant Naming Convention Use ALL_CAPS for constants so they stand out from regular variables. This is a widely-used convention.

Variable Scope — Where Can It Be Used?

Option Explicit ' ── MODULE LEVEL (all subs in this module can use it) ── Dim moduleVar As String ' ── PUBLIC (all modules in the workbook can use it) ── Public globalCounter As Long Sub Sub1() ' ── LOCAL (only exists inside this Sub) ── Dim localVar As Integer localVar = 10 moduleVar = "Hello" End Sub Sub Sub2() ' localVar is NOT available here — it died when Sub1 ended ' moduleVar IS available here MsgBox moduleVar End Sub
📐 Rule of Thumb Use the narrowest scope possible. Local variables (inside Sub) are safest. Avoid Public unless truly needed.

Working with Strings

Dim s1 As String, s2 As String, result As String s1 = "Yamaha" s2 = " Motor" ' Concatenation (joining strings) — use & result = s1 & s2 ' → "Yamaha Motor" ' String length MsgBox Len(s1) ' → 6 ' Convert case MsgBox UCase(s1) ' → "YAMAHA" MsgBox LCase(s1) ' → "yamaha" ' Extract part of a string MsgBox Left(s1, 3) ' → "Yam" MsgBox Mid(s1, 2, 3) ' → "ama" (from pos 2, 3 chars) ' Convert number to string and back Dim n As Long: n = 42 MsgBox CStr(n) ' → "42" (number → string) MsgBox CLng("100") ' → 100 (string → number)

✏️ Practice Exercise

Exercise: Personal Info Macro

Write a macro that:

  1. Declares variables: name (String), age (Integer), salary (Double)
  2. Assigns values to each variable
  3. Displays: "Name: [name], Age: [age], Salary: ¥[salary]"
  4. Writes the info to cells A1:C1 on the active sheet

💡 Solution

Option Explicit Sub PersonalInfo() Dim empName As String Dim empAge As Integer Dim empSalary As Double empName = "Tanaka Hiroshi" empAge = 28 empSalary = 350000.0 MsgBox "Name: " & empName & ", Age: " & empAge & ", Salary: ¥" & empSalary Range("A1").Value = empName Range("B1").Value = empAge Range("C1").Value = empSalary End Sub
