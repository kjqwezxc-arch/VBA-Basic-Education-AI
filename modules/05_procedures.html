Procedures & Functions

Organize your code into reusable building blocks — Sub, Function, parameters, scope

Sub vs. Function — What's the Difference?

🔧 Sub (Subroutine)

  • Performs actions (no return value)
  • Called with Call MySub or just MySub
  • Can be assigned to buttons, shortcuts
  • Good for: formatting, writing data, showing dialogs
Sub FormatHeader() Range("A1").Font.Bold = True Range("A1").Font.Size = 14 End Sub

📊 Function

  • Calculates and returns a value
  • Can be used in Excel cells like a formula!
  • Called: result = MyFunction(args)
  • Good for: calculations, data transformations
Function AddTax(price As Double) As Double AddTax = price * 1.10 End Function
📌 Key RuleIf you need a result back → use Function. If you just want to do something → use Sub.

Parameters — Passing Data to Procedures

' Sub with parameters Sub HighlightRange(targetRange As Range, highlightColor As Long) targetRange.Interior.Color = highlightColor End Sub ' Calling the sub with arguments: HighlightRange Range("A1:A10"), RGB(255, 255, 0) ' Function with multiple parameters Function CalcBonus(salary As Double, pctRate As Double) As Double CalcBonus = salary * (pctRate / 100) End Function ' Calling it: Dim myBonus As Double myBonus = CalcBonus(450000, 15) ' → 67500 MsgBox "Bonus: ¥" & myBonus

ByVal vs. ByRef — Parameter Passing

ByRef (default) — passes the actual variable. Changes inside the Sub affect the original!

Sub DoubleIt(n As Long) ' ByRef (default) n = n * 2 End Sub Dim x As Long: x = 5 DoubleIt x MsgBox x ' Shows 10! x was changed

ByVal — passes a copy. Changes inside do NOT affect the original.

Sub DoubleIt(ByVal n As Long) ' ByVal n = n * 2 ' only the copy changes End Sub Dim x As Long: x = 5 DoubleIt x MsgBox x ' Still 5! x is unchanged
💡 Best PracticeUse ByVal for most parameters to prevent accidental modification of variables. Use ByRef only when you intentionally want the sub to modify the original variable.

Optional Parameters & Default Values

' Optional parameters have defaults Function FormatCurrency(ByVal amount As Double, _ Optional currency As String = "¥", _ Optional decimals As Integer = 0) As String FormatCurrency = currency & Format(amount, String(decimals, "0") _ & IIf(decimals>0,"."&String(decimals,"0"),"")) End Function ' All these are valid calls: MsgBox FormatCurrency(50000) ' → "¥50000" MsgBox FormatCurrency(50000, "$") ' → "$50000" MsgBox FormatCurrency(50000.5, "¥", 2) ' → "¥50000.50" ' Check if optional parameter was passed: Function Greet(Optional name As String = "") As String If IsMissing(name) Or name = "" Then Greet = "Hello, World!" Else Greet = "Hello, " & name & "!" End If End Function

Procedure Scope — Public vs. Private

'═══ MODULE 1 ═══════════════════════════════ ' PUBLIC: Callable from anywhere in the workbook Public Sub GenerateReport() FormatData() ' Can call private subs in same module ExportToPDF() End Sub ' PRIVATE: Only callable from this module Private Sub FormatData() ' Internal helper — not visible outside module End Sub '═══ MODULE 2 ═══════════════════════════════ Sub TestCalls() GenerateReport() ' ✅ Works (Public) ' FormatData() ← ❌ Error! (Private) End Sub
📌 Scope Best PracticesMake all procedures Private by default. Only make them Public when they need to be called from other modules or from Excel buttons.

User-Defined Functions (UDFs) in Excel

VBA Functions can be used directly in Excel cells like built-in functions!

' Add this function to a Module Function TaxIncluded(ByVal price As Double) As Double ' Adds 10% tax TaxIncluded = price * 1.10 End Function Function ExtractNumbers(ByVal text As String) As String Dim i As Long, result As String For i = 1 To Len(text) If Mid(text, i, 1) >= "0" And Mid(text, i, 1) <= "9" Then result = result & Mid(text, i, 1) End If Next i ExtractNumbers = result End Function
✅ Using UDFs in ExcelIn any cell, type =TaxIncluded(A1) or =ExtractNumbers(B2) — just like VLOOKUP or SUM!

Calling Procedures from Other Procedures

Option Explicit ' ── HELPER FUNCTIONS ─────────────────────── Private Function IsDataValid(ByVal ws As Worksheet) As Boolean IsDataValid = ws.Cells(1, 1) <> "" End Function Private Sub FormatReport(ByVal ws As Worksheet) With ws.Range("A1:Z1") .Font.Bold = True .Interior.Color = RGB(21, 101, 192) .Font.Color = RGB(255,255,255) End With End Sub ' ── MAIN PROCEDURE ───────────────────────── Public Sub GenerateMonthlyReport() Dim ws As Worksheet Set ws = ThisWorkbook.Worksheets("Data") If Not IsDataValid(ws) Then MsgBox "No data found!": Exit Sub End If FormatReport ws MsgBox "Report generated!" End Sub
