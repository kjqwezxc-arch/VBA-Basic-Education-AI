Error Handling & Debugging

Write robust macros that handle errors gracefully, and debug like a pro

Types of Errors in VBA

⚡ Syntax Errors

Code that doesn't follow VBA grammar rules. Caught immediately as you type.

If x = 5 ' Missing Then Dim x As ' Incomplete

VBA underlines these in red automatically.

💥 Runtime Errors

Code runs but crashes during execution. Most common type.

' Division by zero x = 10 / 0 ' Sheet doesn't exist Sheets("Bad").Select ' Type mismatch x = CInt("abc")

🔍 Logic Errors

Code runs without crashing but produces wrong results. Hardest to find!

' Wrong: should start at 2 For i = 1 To lastRow ' Skips header row fix For i = 2 To lastRow

On Error GoTo — Catching Errors

Sub SafeMacro() On Error GoTo ErrorHandler ' Register error handler FIRST ' Normal code here Dim ws As Worksheet Set ws = Sheets("Report") ' May fail if sheet missing Dim result As Double result = Cells(1,2).Value / Cells(1,3).Value ' May divide by zero MsgBox "Result: " & result Exit Sub ' IMPORTANT: Skip the error handler when no error ErrorHandler: MsgBox "Error #" & Err.Number & ": " & Err.Description, _ vbCritical, "Macro Error" ' Log the error or clean up here End Sub
📌 Always Exit Sub before ErrorHandlerWithout Exit Sub, VBA would always run the error handler code even when there's no error!

On Error Resume Next — Ignore & Continue

' ON ERROR RESUME NEXT — continues execution after an error Sub CheckSheetExists() Dim ws As Worksheet On Error Resume Next Set ws = Sheets("Data") ' Try to set — might fail On Error GoTo 0 ' RESTORE normal error handling immediately! If ws Is Nothing Then MsgBox "Sheet 'Data' not found!" Else MsgBox "Sheet found: " & ws.Name End If End Sub
⚠️ Warning: Use Sparingly! On Error Resume Next can hide real bugs. Always restore with On Error GoTo 0 as soon as possible after the risky operation.
' The Err object — get details about the last error If Err.Number <> 0 Then Debug.Print "Error: " & Err.Number & " - " & Err.Description Err.Clear ' Reset the Err object End If

The Err Object & Resume

Err Object Properties

' Common Err properties Err.Number ' Error number (0=no error) Err.Description ' Error message text Err.Source ' Where error came from Err.Clear ' Reset error info ' Common error numbers: ' 9 = Subscript out of range ' 11 = Division by zero ' 13 = Type mismatch ' 91 = Object variable not set ' 1004 = Application-defined error

Resume Variations

' Resume — retry the line that failed ErrorHandler: If Err.Number = 11 Then Cells(1,3).Value = 1 ' Fix the problem Resume ' Try again End If ' Resume Next — skip the failing line ErrorHandler: Debug.Print "Error: " & Err.Description Resume Next ' Continue from next line ' Resume [label] — jump to a specific label ErrorHandler: MsgBox "Failed" Resume Cleanup

Debugging Tools in the VBE

🔴 Breakpoints (F9)

Click the left margin next to a line (or press F9) to set a red breakpoint. Code stops there so you can inspect values.

Click margin → red dot appears → run macro → code pauses at that line

👟 Step Through (F8)

Press F8 to run one line at a time. Hover over variables to see their current value. Use Step Over (Shift+F8) to skip into sub-calls.

F8 = Step Into   Shift+F8 = Step Over   Ctrl+Shift+F8 = Step Out

👁️ Watch Window

Right-click a variable → Add Watch. VBE shows its value in real time as you step through code.

View → Watch Window → Right-click any variable → "Add Watch"
💡 Immediate Window (Ctrl+G)Type ? variableName and press Enter to instantly check a value. Also run single lines of code directly. Essential for debugging!

Professional Error Handling Pattern

Sub RobustMacro() On Error GoTo ErrorHandler ' Turn off for performance Application.ScreenUpdating = False Application.EnableEvents = False ' === Your main code here === Dim ws As Worksheet Set ws = Sheets("Data") ' ... do work ... MsgBox "Complete!", vbInformation Cleanup: ' ALWAYS restore settings — runs whether error or not Application.ScreenUpdating = True Application.EnableEvents = True Exit Sub ErrorHandler: Dim errNum As Long Dim errMsg As String errNum = Err.Number errMsg = Err.Description MsgBox "Error " & errNum & ": " & errMsg & _ Chr(13) & "Please check your data and try again.", _ vbCritical, "Macro Error" Resume Cleanup ' Always clean up, even after error End Sub
