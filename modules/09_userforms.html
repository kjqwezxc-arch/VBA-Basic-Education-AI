UserForms & UI

Build professional dialog boxes and data-entry forms with VBA

What is a UserForm?

A UserForm is a custom dialog box you design visually in the VBE. It replaces multiple InputBox/MsgBox calls with a professional interface.

✅ Why use UserForms? • Collect multiple inputs at once
• Input validation before processing
• Professional look for your tools
• Dropdown lists, checkboxes, buttons
• Reusable across projects
📌 How to createVBE → Insert menu → UserForm → Design visually using Toolbox → Write event code behind the form

Creating a UserForm — Step by Step

Step 1: Add the Form

VBE → Insert → UserForm
Toolbox panel appears automatically

Step 2: Set Properties

Click the form → Properties window (F4)
Change Name, Caption, Width, Height

Step 3: Add Controls

Drag controls from Toolbox onto form
Click each control → set properties in F4

Step 4: Write Code

Double-click a button → VBE creates the Click event procedure for you

Button Events — Reading & Validating Input

' Submit button click event Private Sub btnSubmit_Click() ' 1. Validate input first If Trim(txtName.Text) = "" Then MsgBox "Please enter a name!", vbExclamation txtName.SetFocus ' Move cursor back to the field Exit Sub End If If cboDepart.Value = "" Then MsgBox "Please select a department!", vbExclamation Exit Sub End If ' 2. Write data to next empty row Dim ws As Worksheet Set ws = Sheets("Data") Dim nextRow As Long nextRow = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row + 1 ws.Cells(nextRow, 1).Value = txtName.Text ws.Cells(nextRow, 2).Value = cboDepart.Value ws.Cells(nextRow, 3).Value = chkFT.Value ws.Cells(nextRow, 4).Value = Now() MsgBox "Data saved!", vbInformation Unload Me ' Close the form End Sub Private Sub btnCancel_Click() Unload Me ' Close without saving End Sub

Showing & Hiding Forms

' Show a form from a regular macro Sub OpenDataEntry() UserForm1.Show ' Modal (default) End Sub ' Show as modeless (non-blocking) UserForm1.Show vbModeless ' Load (prepare) without showing Load UserForm1 UserForm1.txtName.Text = "Default" UserForm1.Show ' Hide (keep in memory) UserForm1.Hide ' Unload (remove from memory) Unload UserForm1 ' Close on X button click Private Sub UserForm_QueryClose( _ Cancel As Integer, _ CloseMode As Integer) If CloseMode = vbFormControlMenu Then Cancel = True ' Prevent X-close MsgBox "Use Cancel button!" End If End Sub

Modal vs Modeless

Modal (default) — user MUST respond to the form before clicking anything else in Excel. Best for data entry.

Modeless — user can interact with Excel while the form is open. Good for toolbars/panels.

📌 Unload vs HideUse Hide to keep the form data in memory. Use Unload to fully remove the form and free memory.

Key Form Events

' TextBox Change — real-time validation Private Sub txtQty_Change() If IsNumeric(txtQty.Text) Then lblError.Caption = "" lblError.ForeColor = RGB(0,128,0) Else lblError.Caption = "⚠ Must be a number" lblError.ForeColor = RGB(255,0,0) End If End Sub ' ComboBox Change — update related fields Private Sub cboProduct_Change() ' Lookup price when product is selected Dim price As Variant On Error Resume Next price = Application.WorksheetFunction.VLookup( _ cboProduct.Value, Sheets("Prices").Range("A:B"), 2, False) On Error GoTo 0 txtPrice.Text = IIf(IsEmpty(price), "N/A", price) End Sub ' OptionButton — respond to radio selection Private Sub optFull_Click() txtHours.Enabled = False ' Disable hours for full-time End Sub Private Sub optPart_Click() txtHours.Enabled = True ' Enable for part-time End Sub
