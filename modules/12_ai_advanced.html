Prompt Engineering — Good vs Great

The difference between a vague and specific prompt is often the difference between broken and perfect code:

❌ VAGUE — likely gets generic, non-working code
"Write a macro to process my Excel data"
⚠️ OK — better but missing key details
"Write a macro to copy rows from Sheet1 to Sheet2 if column A contains 'Approved'"
✅ GREAT — specific, contextual, constrained
"Act as a VBA expert. I have Sheet1 with data starting from row 2 (row 1 = headers: Order ID, Status, Amount, Date in columns A–D). Write a macro that copies all rows where column B (Status) = 'Approved' to a sheet named 'Approved Orders', creating that sheet if it doesn't exist. Clear the destination sheet before copying. Include error handling, add comments, and use Long for row variables."
🎯 The 3 Levels of Specificity Data structure — sheet names, column positions, header rows
Exact behavior — what happens in edge cases (empty, already exists, etc.)
Code quality — error handling, comments, variable types

Advanced: Role + Context Injection

Give AI a specific role and deep context about your project before asking the main question:

🎭 Role Injection Template "You are a senior Excel VBA developer who specializes in automated reporting for manufacturing companies. I am developing a macro-enabled workbook for a motorcycle company's sales team. The workbook has these sheets: [list them]. Column structure for each sheet: [describe]. Our coding standards: always use error handling with On Error GoTo, use descriptive variable names, add comments above each block of logic, declare all variables with Dim, and use ThisWorkbook instead of ActiveWorkbook. [Now ask your question]"

Without Role Injection

Write a macro to send a monthly report
Generic code with placeholder names like "Sheet1", no error handling, uses ActiveWorkbook...

With Role Injection

[After setting the context above] Write a macro to send the monthly report
Code uses your exact sheet names, includes proper error handling, ThisWorkbook, your naming conventions, and relevant comments...

Iterative Refinement — One Step at a Time

Don't try to ask for everything at once. Build up the macro through a conversation:

Step 1: Write just the skeleton of a macro called ProcessSalesData that opens a file and reads all rows
Here's the skeleton: [code with structure but no filtering logic]
Step 2: Good. Now add filtering: only process rows where column C > 1000
Updated code with If Cells(i,3).Value > 1000 Then filter added...
Step 3: Now add code to write results to a summary sheet and auto-format the output as a table
Final complete macro with all three features integrated...
✅ Why Iterative Works Better • Each step is testable — you catch errors early
• AI doesn't lose context building on prior responses
• Easier to understand what each part does
• You can stop and redirect at any point

AI Code Review — Find Issues Before They Happen

Before deploying a macro to colleagues, ask AI to review it:

Please review this VBA macro for: 1. Bugs or logic errors 2. Missing error handling 3. Performance issues 4. Anything that could fail on large datasets or edge cases [paste your code]
Code Review Report: 🔴 Bug found (Line 12): Loop uses integer i but row count may exceed 32,767. Change i to Long. 🟡 Missing error handling: The Sheets("Report") reference on Line 8 will crash if that sheet is renamed. Add: On Error Resume Next / Set ws = Sheets("Report") / On Error GoTo 0 🟡 Performance: ScreenUpdating is never turned off. For large datasets add Application.ScreenUpdating = False at start. 🟢 Logic: The backwards delete loop (Step -1) is correct — no issues found. ✅ Overall: Functional but needs the Integer→Long fix before running on large files.

Building a Complete Application with AI

Use AI to build a full macro-based tool through a structured conversation:

Define requirements — "I need a VBA tool that does X, Y, Z. Here's the sheet structure..."
Get architecture plan — "What Subs/Functions do I need? Describe the structure first."
Build module by module — "Write just the data validation module first."
Test each piece — run in Excel, note any issues, report back to AI.
Integrate — "Now connect all modules into a main entry Sub that calls each step."
Final review — "Review the complete code for any issues and add missing error handling."
🏗️ Example: Build a Full Reporting ToolWith this approach, you can build a complete automated report generator in 1–2 hours that would have taken days without AI. The key: YOU guide the build process; AI writes the code.

AI Limitations — Know the Boundaries

⚠️ What AI Gets Wrong
  • 🔴 Your actual file — AI can't see your workbook. It guesses your structure.
  • 🔴 Business rules — AI doesn't know your team's special requirements
  • 🔴 Security — AI may suggest storing passwords as plain text strings — never do this
  • 🔴 Latest Excel features — AI knowledge may be outdated
  • 🔴 Confidential data — never paste real customer/employee data into AI chats
✅ What AI Does Very Well
  • 🟢 Writing boilerplate/repetitive code
  • 🟢 Converting pseudocode to working VBA
  • 🟢 Explaining unfamiliar syntax
  • 🟢 Diagnosing errors from messages
  • 🟢 Suggesting alternative approaches
  • 🟢 Optimizing slow code
  • 🟢 Adding error handling to existing code
  • 🟢 Translating requirements to code
🔒 Privacy RulesWhen using AI for work: describe your data structure but use sample/fake data in examples. Never include real names, IDs, or business-sensitive numbers in your AI prompts.

The Human + AI Partnership

You now have both VBA knowledge AND AI skills. Here's how they combine:

🤖 What AI Brings

  • • Speed — writes code faster than you can type
  • • Breadth — knows thousands of VBA patterns
  • • Patience — happy to explain the same thing many ways
  • • Memory — never forgets syntax

🧑 What You Bring

  • • Context — you know your data, your team's needs
  • • Judgment — you can tell if code makes sense
  • • Validation — you can test and verify results
  • • Business rules — you understand the "why"
🚀 The Result A skilled developer with AI assistance can build tools 5–10× faster than without AI — while maintaining quality because they can validate the output. This is the new skill set for modern office automation.
