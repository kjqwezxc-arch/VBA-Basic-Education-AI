Writing Good VBA Prompts

The quality of your AI output depends entirely on how well you describe what you want. Follow the 4-Part Prompt Formula:

1. Role

"Act as a VBA expert" / "You are an Excel macro programmer"

2. Task

"Write a VBA macro that..." — be specific about WHAT it should do

3. Context

"The data is in Sheet1, column A has names, column B has values, row 1 is the header"

4. Format / Constraints

"Include error handling, add comments, use descriptive variable names"

✅ Example of a Great Prompt"Act as an Excel VBA expert. Write a macro that reads data from Sheet1 (headers in row 1: Name, Sales, Target in columns A-C), calculates if Sales exceeds Target, and highlights the row green if yes, red if no. Include error handling and comments."

Write a VBA macro that copies all rows where column C contains "Pending" to a new sheet called "Pending Items". Include error handling.
Here's a VBA macro that does exactly that: Sub CopyPendingItems() On Error GoTo ErrorHandler Dim srcWs As Worksheet, dstWs As Worksheet Set srcWs = ThisWorkbook.ActiveSheet ' Create or clear the destination sheet On Error Resume Next Set dstWs = ThisWorkbook.Sheets("Pending Items") On Error GoTo ErrorHandler If dstWs Is Nothing Then Set dstWs = ThisWorkbook.Sheets.Add dstWs.Name = "Pending Items" Else dstWs.Cells.Clear End If ' Copy header srcWs.Rows(1).Copy dstWs.Rows(1) ' Copy matching rows Dim lastRow As Long, dstRow As Long lastRow = srcWs.Cells(srcWs.Rows.Count, 3).End(xlUp).Row dstRow = 2 Dim i As Long For i = 2 To lastRow If srcWs.Cells(i, 3).Value = "Pending" Then srcWs.Rows(i).Copy dstWs.Rows(dstRow) dstRow = dstRow + 1 End If Next i MsgBox (dstRow - 2) & " rows copied.", vbInformation Exit Sub ErrorHandler: MsgBox "Error: " & Err.Description, vbCritical End Sub

Using AI to Explain Existing Code

Not sure what a piece of code does? Ask AI to explain it!

Explain this VBA code step by step: Dim lastRow As Long lastRow = Cells(Rows.Count, 1).End(xlUp).Row For i = lastRow To 2 Step -1 If Cells(i, 3).Value = "" Then Rows(i).Delete End If Next i
This macro deletes rows with empty values in column C, working from the bottom up: 1. lastRow — finds the last used row in column A 2. For i = lastRow To 2 Step -1 — loops backwards (bottom to top). This is important! If you delete rows going forward, the row numbers shift and you skip rows. 3. If Cells(i, 3).Value = "" — checks if column C (index 3) is empty 4. Rows(i).Delete — deletes the entire row if empty 5. The loop goes down to row 2 (skipping row 1, which is the header) The backwards loop pattern is a common VBA technique when deleting rows.

Common Mistakes AI Makes with VBA

❌ Mistakes to Watch For
  • 🔴 Wrong sheet reference — uses ActiveSheet when you need a specific sheet
  • 🔴 Missing error handling — assumes files/sheets always exist
  • 🔴 Hardcoded row counts — uses 1000 instead of finding actual last row
  • 🔴 Wrong object library — may generate code requiring references not installed
  • 🔴 Outdated syntax — occasionally uses deprecated approaches
  • 🔴 Logic errors — compiles fine but wrong results in edge cases
✅ How to Validate AI Code
  1. Read it before running — do you understand what it does?
  2. Run on a copy of your data first
  3. Check sheet/range references match your actual structure
  4. Check error handling is present
  5. Test with edge cases (empty cells, 1 row, 1000 rows)
  6. Ask AI: "Are there edge cases this doesn't handle?"

Ask AI to Debug Your Errors

When your macro produces an error, paste the error message + code into AI and ask for help:

My VBA macro gives "Runtime error 9: Subscript out of range" on this line: Set ws = Sheets("Sales Data") Here's the full macro: [paste your code] How do I fix this?
Error 9 "Subscript out of range" means the sheet named "Sales Data" doesn't exist in the workbook. Common causes: 1. Typo in sheet name — check the exact spelling and spacing (including capital letters) 2. Sheet doesn't exist yet — the sheet hasn't been created 3. Extra spaces — the sheet might be named " Sales Data" with a leading space Fix options: • Check the exact sheet tab name and update your code to match • Add a SheetExists check before setting: On Error Resume Next Set ws = Sheets("Sales Data") On Error GoTo 0 If ws Is Nothing Then MsgBox "Sheet not found!"
🤖 Pro Tip: Include ContextAlways paste the full error message, the line it occurred on, and the surrounding code. The more context you give AI, the better the diagnosis.
